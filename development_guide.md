# 微信小游戏开发详细流程

本指南将为您详细介绍如何从零开始开发一款微信小游戏。我们将以您当前的“推箱子”小游戏为例，讲解从项目搭建到最终上线的完整流程。

## 1. 准备工作

### 第一部分：开发环境与账号

在开始敲代码之前，请确保您已准备好以下环境和工具：

*   **微信开发者工具**: 这是微信官方提供的集成开发环境（IDE），用于小游戏和公众号的开发、调试和发布。请从 [微信开放平台](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html) 下载并安装最新版本。
*   **微信账号与 AppID**:
    *   **个人微信账号**: 用于登录开发者工具。
    *   **AppID**: 游戏的唯一身份证。开发初期可使用开发者工具提供的“测试号”，但游戏上线前必须前往 [微信公众平台](https://mp.weixin.qq.com/) 注册小程序账号以获取正式 AppID。
*   **技术基础**:
    *   **JavaScript (ES6+)**: 编写游戏逻辑的核心语言，需要熟悉 `class`, `import/export`, `async/await` 等特性。
    *   **Canvas API**: 了解如何使用 Canvas API 在画布上进行绘制，这是小游戏渲染的基础。
    *   **HTML/CSS 概念**: 有助于理解布局、定位和颜色等视觉概念。

### 第二部分：游戏内容与素材

这部分准备工作决定了游戏的“血肉”，直接影响品质和风格。

*   **游戏视觉素材 (Art Assets)**: 玩家在游戏中所看到的一切。
    *   **内容清单**:
        *   **角色/单位 (Sprites):** 玩家、敌人等，可能需要包含不同状态（站立、跑动）的序列帧。
        *   **场景元素 (Tilesets):** 用于拼接地图的地板、墙壁、草地等基础瓦片。
        *   **游戏对象 (Objects):** 宝箱、道具、障碍物等。
        *   **背景图 (Backgrounds):** 营造游戏氛围的远景。
        *   **UI 元素 (UI Elements):** 按钮、图标、面板、对话框等。
        *   **特效 (VFX):** 爆炸、点击效果、光芒等序列帧动画。
    *   **获取方式**:
        *   **原创**: 使用 Photoshop, Aseprite (像素画), Krita (免费) 等工具自行绘制。
        *   **免费资源网站**: Kenney.nl, itch.io, OpenGameArt.org 是很好的选择。**注意：使用前务必检查并遵守其授权协议 (License)。**
        *   **付费/外包**: 从专业的素材商店购买或寻找美术设计师定制。

*   **文字与字体 (Text & Fonts)**:
    *   **系统字体**: 最简单的方案，直接在代码中调用（如 `Arial`），但样式单一且跨平台表现不一。
    *   **自定义字体**:
        *   **来源**: Google Fonts, 以及“站酷”、“字由”等中文网站提供大量免费可商用字体。**注意：字体有版权，上线前必须确认授权！**
        *   **使用**: 将字体文件（.ttf, .woff 等）放入项目，通过 `wx.loadFont()` API 加载后即可在代码中使用。
    *   **图片字体 (Bitmap Fonts)**: 将文字设计成图片，适用于标题、Logo 等无需动态改变内容的艺术字。

*   **页面与UI布局 (Pages & UI Layout)**:
    *   **流程规划**: 使用流程图画出所有页面的跳转关系（如：启动页 -> 主菜单 -> 游戏场景 -> 结束页）。
    *   **界面草图 (Wireframe)**: 用简单的线条和方框，快速规划每个页面“什么东西放在什么位置”。
    *   **视觉设计稿 (Mockup)**: 使用 Figma, Sketch, Photoshop 等工具，基于草图完成精细的视觉设计，确定最终的颜色、样式和布局。

## 2. 项目结构解析

一个典型的小游戏项目包含以下几个核心部分。我们将以您的“推箱子”项目为例进行说明：

### 根目录

*   `game.js`: 微信小游戏的入口文件，负责初始化游戏。
*   `game.json`: 微信小游戏的配置文件，定义了小游戏的窗口表现、屏幕方向（横屏/竖屏）等。
*   `project.config.json`: 微信开发者工具的项目配置文件，包含了项目设置、AppID 等信息。

### `js/` - 核心逻辑目录

这是存放游戏核心 JavaScript 代码的地方。

*   `main.js`: 游戏主循环和入口，负责创建 canvas、启动游戏循环、监听用户输入和更新游戏状态。
*   `game_scene.js`: 游戏主场景，负责初始化关卡、渲染所有实体、处理游戏逻辑。
*   `databus.js`: 全局状态管理器，用于在游戏的不同模块间共享数据。
*   `resources.js`: 定义了需要加载的图片资源列表。
*   **`entities/`**: 存放游戏中所有“实体”的定义，每个文件对应一种游戏元素（如 `player.js`, `box.js`）。

### `images/` - 资源目录

存放游戏所需的所有静态图片资源，例如 `player.png`, `box.png` 等。

### `levels/` - 关卡数据

存放游戏的关卡数据，通常以 JSON 格式存储。

## 3. 开发流程详解

### 第一步：编写核心代码

#### **游戏入口 (`game.js`)**

*   这是小游戏的主入口，您需要在这里引入主循环文件 `js/main.js`。

```javascript
import './js/libs/adapter';
import './js/main';
```

#### **主循环和场景 (`js/main.js` 和 `js/game_scene.js`)**

*   在 `main.js` 中，您需要：
    1.  创建 Canvas 画布。
    2.  加载所有必要的资源（图片、音频等）。
    3.  初始化游戏场景 (`GameScene`)。
    4.  启动游戏主循环 (`gameLoop`)。
*   `gameLoop` 是游戏的核心，它在每一帧都会被调用，负责：
    1.  清空画布。
    2.  更新游戏状态（例如，玩家移动、碰撞检测）。
    3.  重新绘制所有游戏元素。
    4.  通过 `requestAnimationFrame` 实现循环调用。

#### **实体和组件 (`js/entities/`)**

*   为游戏中的每一个对象（玩家、敌人、道具）创建一个单独的 JavaScript 类。
*   每个类都应该包含该对象的属性（例如，位置 `x` 和 `y`）和方法（例如，`update()` 和 `draw()`）。

### 第二步：资源管理

*   在 `js/resources.js` 中，以键值对的形式定义好所有需要加载的图片资源。
*   在游戏开始前，使用 `resourceLoader.js` 中的逻辑来预加载所有资源，确保游戏过程中不会因为资源加载而出现卡顿。

### 第三步：关卡设计

*   在 `levels/` 目录中，使用 JSON 文件来设计您的游戏关卡。
*   每个 JSON 文件代表一个关卡，通常用一个二维数组来表示地图布局。

### 第四步：调试和预览

*   **模拟器**: 微信开发者工具内置了模拟器，可以模拟不同型号的手机设备，方便您测试游戏的兼容性。
*   **真机调试**: 这是最重要的调试方式。点击开发者工具顶部的“真机调试”按钮，用微信扫描生成的二维码，即可在您的手机上实时预览和调试游戏。

## 4. 上线发布

当您的游戏开发完成并通过充分测试后，就可以准备上线了。

1.  **提交审核**: 在开发者工具中，点击右上角的“上传”按钮，填写版本信息，然后将您的游戏代码上传到微信后台。
2.  **登录微信公众平台**: 访问 [微信公众平台](https://mp.weixin.qq.com/)，在“开发管理”中找到您刚刚上传的版本。
3.  **提交审核**: 点击“提交审核”，并根据要求填写相关信息。
4.  **等待审核**: 微信团队会对您的游戏进行审核，确保其内容和功能符合平台规范。
5.  **发布上线**: 审核通过后，您可以选择“手动发布”或“自动发布”。发布后，所有微信用户就都可以在微信中搜索并玩到您的游戏了。
